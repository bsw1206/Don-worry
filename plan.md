## 1. 서비스 정의 및 개발 범위 확정

### 서비스 한 줄 정의
> "사용자의 소비 패턴·소득 수준·위험 성향에 맞는 금융상품을 비교하고 추천받는 웹 서비스"

### 제공 채널
- **PC 웹 브라우저 우선** / 모바일 브라우저에서 레이아웃이 깨지지 않는 반응형 디자인 적용
- 별도 모바일 앱 개발 없음

---

### ✅ 1순위 — 핵심 기능 (반드시 완성)

저축 상품 영역과 투자 상품 영역을 분리하여 운영한다.
ETF 페이지는 로그인 사용자라면 누구나 접근할 수 있으며, 위험도 점수는 접근 차단이 아닌 맞춤 추천 기준으로만 활용한다.
단, 종목명에 '레버리지'가 포함된 ETF는 금융투자교육원 이수 여부 확인 후 관심 상품 추가가 가능하다.

| 기능 | 사용 API | 비고 |
|------|----------|------|
| 회원가입 / 로그인 | — | JWT 인증 |
| **투자 위험도 설문** | — | 사전 정의 점수 합산 방식, 저축·ETF 추천의 개인화 기준 |
| 정기예금 목록·검색·상세 | Finlife API | |
| 적금 목록·검색·상세 | Finlife API | |
| 연금저축 목록·검색·상세 | Finlife API | |
| 저축 상품 맞춤 추천 | Finlife API | 위험도 점수 + 설문 답변 기반 |
| ETF 목록·검색·상세 | 한국투자증권 API | 로그인 사용자 전체 접근 가능 |
| ETF 맞춤 추천 | 한국투자증권 API | 위험도 점수를 추천 개인화에 활용 |
| 실시간 시장 요약 | 한국투자증권 API | 지수/환율/채권/원자재, ETF 페이지 내 노출 |
| 관심 상품 저장 (찜) | — | 저축 상품·ETF 통합 관리 / 레버리지 ETF는 교육 이수 확인 후 추가 |

### 🔵 2순위 — 부가 기능 (여유 시 추가)

| 기능 | 사용 API | 전제 조건 |
|------|----------|-----------|
| 실시간 거래 현황 | 한국투자증권 API | 순매수/순매도, 기관·개인·외국인별 상위 5개, 등락률·거래 규모 |
| 실시간 차트 | 한국투자증권 API | ETF 종목별 시세 차트 |

### ❌ 개발 범위 제외

- 상품 가입·결제·거래 기능 (추천 전용 서비스)
- 펀드 상품 (데이터 수급 방법 미확정)
- 실시간 시세 자동 갱신 (수동 갱신 또는 페이지 로드 시 1회 호출)
- 커뮤니티·댓글
- 소셜 로그인
- 실제 보유 자산 연동

## 2. 데이터 출처 및 수집 구조

### 2-1. 금융감독원 Finlife API (1순위)

저축 상품 데이터의 유일한 출처. Django management command로 DB에 저장하여 활용한다.

| 상품 유형 | API 명칭 | 구조 | 수집 방식 |
|-----------|----------|------|-----------|
| 정기예금 | `fdrmDpstApi` | baseList + optionList | 초기 1회 DB 저장, 이후 수동 갱신 |
| 적금 | `fdrmEntyApi` | baseList + optionList (rsrv_type 추가) | 동일 |
| 연금저축 | `annuSavApi` | 별도 필드 구조 | 동일 |

**Finlife API 주요 공통 필드**

| 필드명 | 설명 |
|--------|------|
| `fin_co_no` | 금융회사 코드 |
| `kor_co_nm` | 금융회사명 |
| `fin_prdt_cd` | 금융상품 코드 |
| `fin_prdt_nm` | 금융상품명 |
| `join_way` | 가입 방법 (인터넷/영업점 등) |
| `join_member` | 가입 대상 |
| `spcl_cnd` | 우대금리 조건 |
| `etc_note` | 유의사항 |
| `max_limit` | 최대 가입 금액 |
| `intr_rate` | 기본금리 (%) |
| `intr_rate2` | 최고금리 (%) |
| `save_trm` | 가입 기간 (개월) |
| `rsrv_type` | 적립 방식 — 적금 전용 (S: 정액, F: 자유) |

---

### 2-2. 한국투자증권 Open API (1순위)

ETF 종목 정보와 실시간 시장 요약 정보를 제공한다.
API 포털: https://apiportal.koreainvestment.com/intro

#### 필수 활용 기능

| 기능 | 설명 | 활용 위치 |
|------|------|-----------|
| **종목 정보 조회** | ETF 종목명, 현재가, 기초지수, 총보수, 거래량, 시가총액 등 | ETF 목록·상세 화면 |
| **실시간 시장 요약** | 주요 지수(코스피/코스닥), 환율, 채권 금리, 원자재 현황 | ETF 페이지 상단 고정 위젯 |

#### 2순위 활용 기능 (개발 여유 시)

| 기능 | 설명 | 활용 위치 |
|------|------|-----------|
| **실시간 거래 현황** | 기관·개인·외국인별 순매수/순매도 상위 5개, 등락률·거래 규모 | ETF 페이지 하단 섹션 |
| **실시간 차트** | 종목별 시세 시계열 차트 | ETF 상세 화면 탭 |

> ⚠️ 한국투자증권 API는 실시간 데이터를 서버에 저장하지 않고 **페이지 로드 시 직접 호출** 방식으로 활용한다. (DB 저장 불필요)

## 4. 화면별 사용자 경험 상세

### 화면 A — 네비게이션 바 (공통)

```
[ Don-worry 로고 ]   [ 🔍 상품명 또는 은행명 검색 ]   [ ♡ 관심상품 ]   [ 로그인 ]
```

| 요소 | 비로그인 | 로그인 후 |
|------|----------|----------|
| 로고 | 메인 이동 | 메인 이동 |
| 검색창 | 저축 상품 검색 가능 | 저축 상품 + ETF 검색 가능 |
| 관심상품 | 로그인 유도 | 관심상품 화면 이동 |
| 우측 버튼 | "로그인" | 프로필 아이콘 |

---

### 화면 B — 메인(홈)

**목적:** 서비스 가치를 즉시 전달하고 저축 상품 탐색 또는 맞춤 추천으로 유도

#### B-1. 히어로 배너
```
"내 상황에 딱 맞는 금융상품, 한 번에 찾아보세요."
부제: 정기예금 · 적금 · 연금저축 비교 추천

  [ 지금 맞춤 추천 받기 ]   [ 전체 상품 둘러보기 ]
```
- [맞춤 추천 받기]: 비로그인 → 로그인 유도 모달 / 로그인 후 → 설문 화면 이동

#### B-2. 금리 현황 TOP 배너
```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  📌 정기예금  │  │  📌 적금      │  │  📌 연금저축  │
│  OO은행      │  │  XX은행      │  │  ZZ은행      │
│ ★최고 4.50% │  │ ★최고 5.20% │  │ ★최고 3.80% │
│  12개월 기준  │  │  24개월 기준  │  │  5년 이상    │
│  [자세히 보기]│  │  [자세히 보기]│  │  [자세히 보기]│
└──────────────┘  └──────────────┘  └──────────────┘
```
데이터 출처: DB 내 `intr_rate2` 기준 유형별 MAX 조회

#### B-3. 빠른 이자 계산기 (위젯)
```
  예치 금액  [ 1,000,000 ]원    기간  [ 12개월 ▼ ]    [ 계산하기 ]

  → 현재 최고금리(4.50%) 기준
     세전 이자: 45,000원  |  세후 이자 (이자소득세 15.4% 공제): 38,070원
```
- 로그인 없이 누구나 사용 가능
- 계산식: `이자 = 원금 × (금리/100) × (기간/12)` / `세후 = 이자 × (1 - 0.154)`

#### B-4. 추천 상품 미리보기 (로그인 사용자)

| 상태 | 표시 내용 |
|------|-----------|
| 설문 완료 | 저축 상품 추천 카드 3개 + ETF 페이지 안내 배너 (점수 무관 항상 표시) |
| 설문 미완료 | "맞춤 추천을 받으려면 설문을 완료해 주세요" + [설문 시작하기] |
| 비로그인 | 로그인 유도 문구 |

### 화면 C — 회원가입 / 로그인

#### 로그인
- 이메일 + 비밀번호 / JWT 토큰 발급

#### 회원가입
> 기본 정보는 가입 시 입력. 투자 성향 상세 설문은 별도 설문 화면에서 진행.

| 입력 항목 | 입력 방식 | 활용 목적 |
|-----------|-----------|-----------|
| 이메일 | 텍스트 | 로그인 ID |
| 비밀번호 | 비밀번호 | 인증 |
| 닉네임 | 텍스트 | 화면 표시 |
| 나이대 | 선택 (20대 / 30대 / 40대 / 50대 이상) | 연금저축 추천 가중치 |
| 월 소득 수준 | 선택 (200만 미만 / 200~400만 / 400만 이상) | 납입 가능 금액 기준 |

---

### 화면 D — 투자 위험도 설문 & 결과

> 이 설문은 서비스 전체 추천의 기준이 되는 핵심 기능이다.
> 각 선택지에 사전 정의된 점수를 부여하고, 합산 점수로 위험도를 자동 판단한다.
> LLM을 사용하지 않으며, 점수 계산은 서버에서 수행한다.

#### 진입 경로
- 메인 [맞춤 추천 받기] 버튼 (로그인 필수)
- 프로필 화면 [설문 다시 하기]
- 회원가입 완료 직후 자동 유도

#### 설문 구성 및 점수 배분 (예시)

> 총점 기준: **0~14점 → 안정형**, **15~24점 → 중립형**, **25점 이상 → 수익추구형(ETF 진입 기준)**

---

**Q1. 투자 목적은 무엇인가요?** (배점 최대 7점)
| 선택지 | 점수 |
|--------|------|
| ① 원금 보전이 최우선이다 | 1점 |
| ② 안정적인 이자 수익을 원한다 | 3점 |
| ③ 시장 수익률 수준을 기대한다 | 5점 |
| ④ 높은 수익을 위해 손실도 감수할 수 있다 | 7점 |

**Q2. 투자 가능 기간은 얼마나 되나요?** (배점 최대 6점)
| 선택지 | 점수 |
|--------|------|
| ① 6개월 미만 | 1점 |
| ② 6개월~1년 | 2점 |
| ③ 1~3년 | 4점 |
| ④ 3년 이상 | 6점 |

**Q3. 다음 상황에서 어떻게 행동하겠습니까?** (배점 최대 6점)
*(투자한 상품이 20% 손실이 발생했을 때)*
| 선택지 | 점수 |
|--------|------|
| ① 즉시 전액 환매한다 | 1점 |
| ② 일부만 환매한다 | 2점 |
| ③ 유지하며 추이를 지켜본다 | 4점 |
| ④ 오히려 추가 투자한다 | 6점 |

**Q4. 월 저축 가능 금액은 얼마인가요?** (배점 최대 5점)
| 선택지 | 점수 |
|--------|------|
| ① 10만원 미만 | 1점 |
| ② 10~30만원 | 2점 |
| ③ 30~50만원 | 4점 |
| ④ 50만원 이상 | 5점 |

**Q5. 저축 목적은 무엇인가요?** (배점 최대 6점)
| 선택지 | 점수 |
|--------|------|
| ① 비상금·단기 목돈 | 1점 |
| ② 중장기 자산 증식 | 3점 |
| ③ 노후 대비 | 4점 |
| ④ 공격적 자산 증대 | 6점 |

---

#### 점수 계산 및 위험도 판정

```
총점 = Q1 + Q2 + Q3 + Q4 + Q5  (최소 5점, 최대 30점)

판정 기준
  5 ~ 14점  →  안정형

## 5. 핵심 데이터 모델 설계

> Finlife API 응답 구조 기준으로 설계.
> 한국투자증권 API 데이터는 실시간 호출 방식이므로 DB에 저장하지 않는다.
> ETF 관심 상품 저장을 위해 종목 코드(ticker)만 최소한으로 보관한다.

### 5-1. 사용자 모델

```
[User]  ← Django 기본 User 모델 활용
  id, email, password, (username → nickname으로 활용)

[UserProfile]  ← User와 1:1 관계 (OneToOneField)
  user                  : OneToOne → User
  age_group             : CharField     ex) "20s" / "30s" / "40s" / "50s+"
  income_level          : CharField     ex) "low" / "mid" / "high"
  survey_done           : BooleanField  default=False
  risk_score            : IntegerField  NULL=True  ← 설문 완료 후 저장되는 총점
  risk_type             : CharField     NULL=True  ← "safe" / "neutral" / "growth"
                                                   설문 완료 후 자동 판정
  is_leverage_certified : BooleanField  default=False
                                        ← 레버리지 ETF 교육 이수 자가 확인 여부
                                           True이면 이후 레버리지 ETF 찜 시 모달 생략
```

> **핵심 설계 결정 1:** `risk_score`(원시 점수)와 `risk_type`(판정 결과)을 분리 저장한다.
> 점수 기준이 바뀌더라도 원시 점수를 기반으로 재판정이 가능하다.
>
> **핵심 설계 결정 2:** `is_leverage_certified`는 실제 이수 번호를 검증하지 않고
> 사용자의 자가 확인(모달에서 '예' 선택)을 저장하는 플래그다.
> 실제 거래는 본 서비스에서 제공하지 않으므로 이 수준의 확인으로 충분하다.

  ans_q4           : IntegerField  (Q4 선택지 점수)
  ans_q5           : IntegerField  (Q5 선택지 점수)
  total_score      : IntegerField  (합산 총점, 서버에서 자동 계산)
  created_at       : DateTimeField  (auto_now_add=True)
  updated_at       : DateTimeField  (auto_now=True)
```

> `total_score`는 저장 시 서버에서 `ans_q1 + ... + ans_q5`로 계산하여 함께 저장한다.
> `UserProfile.risk_score`와 `UserProfile.risk_type`도 동시에 갱신한다.

---

### 5-6. 관심 상품 모델

```
[Wishlist]
  id           : AutoField   (PK)
  user         : FK → User
  product      : FK → Product  NULL=True   ← 저축 상품 (정기예금/적금/연금저축)
  etf_ticker   : CharField     NULL=True   ← ETF 종목 코드 (한투 API 식별자)
  etf_name     : CharField     NULL=True   ← ETF 종목명 (표시용 캐시)
  saved_at     : DateTimeField  (auto_now_add=True)

  제약: product와 etf_ticker 중 하나는 반드시 존재해야 함 (CHECK 제약 또는 clean() 검증)
       (user, product)  unique_together  ← 저축 상품 중복 저장 방지
       (user, etf_ticker)  unique_together  ← ETF 중복 저장 방지
```

> **설계 근거:** ETF 데이터는 한국투자증권 API에서 실시간 조회하므로 DB에 전체 종목 정보를 저장하지 않는다.
> 찜한 ETF는 `ticker`(종목 코드)와 `etf_name`(표시용)만 저장하고, 상세 정보는 조회 시 API를 호출한다.

### ETF (한국투자증권 API 프록시)

> 클라이언트가 한국투자증권 API를 직접 호출하지 않고 Django 서버를 통해 호출한다.
> (API 키 보호 및 응답 가공 목적)
> ETF 페이지는 로그인 사용자 전체가 접근 가능하다.

| Method | Endpoint | 설명 | 인증 필요 |
|--------|----------|------|-----------|
| GET | `/api/etf/market-summary/` | 실시간 시장 요약 (지수/환율/채권/원자재) | ✅ |
| GET | `/api/etf/` | ETF 목록 (종목 정보) | ✅ |
| GET | `/api/etf/{ticker}/` | ETF 상세 | ✅ |
| GET | `/api/etf/recommend/` | 위험도 점수 기반 ETF 추천 (레버리지 제외) | ✅ |
| POST | `/api/etf/leverage-confirm/` | 레버리지 ETF 교육 이수 자가 확인 → `is_leverage_certified = True` 저장 | ✅ |

**POST `/api/etf/leverage-confirm/` 동작**

```
요청: (바디 없음, 로그인 토큰만 필요)

처리:
  UserProfile.is_leverage_certified = True 로 업데이트

응답:
  { "is_leverage_certified": true }
```

**관심 상품 추가 시 레버리지 ETF 처리 흐름 (서버)**

```
POST /api/wishlist/  (etf_ticker = "레버리지 포함 종목")
  → 서버에서 종목명에 '레버리지' 포함 여부 확인
  → UserProfile.is_leverage_certified == False 이면 403 반환
     { "error": "leverage_not_certified",
       "message": "레버리지 ETF는 금융투자교육원 이수 확인 후 추가할 수 있습니다." }
  → True 이면 정상 저장
```
